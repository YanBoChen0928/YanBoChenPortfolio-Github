    2020年夏天，我在高雄中山大學修習統計學和 R 語言，參考 John Z. 的著作《軟體開發人員職涯發展手冊》並與工程師同學及新朋友討論 CS 領域對我的吸引力與實際面考量，我將焦點轉向程式設計和後端開發，目前先以 Web developer (back-end) 為初始目標，之後再考慮往 software developer 或者 cybersecurity 去深入。

  • Q5 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

    • 以前多位學長跟朋友提到我的個性相當平穩：做事的時候不容易有情緒，我認為這是我自己在大學對自己刻意練習的結果。因為保持情緒的穩定，才可以保有比較清晰的理性思緒與專注。

    另外過去麻醉的工作 9 成的時間跟 case 都相對 peace 、但是約有 1 成時間心力需要全神貫注的處理病人的突發狀況以及手術房裡的急救，也讓我自信有一定的抗壓性。

    因此這個問題我想了一下，我想分享我在踏進新領域時、當找不到具體可以如何加強自己的地方的時候偶爾會有的「內心焦急」（雖然外在不一定會表現出來）。

    比如說我這一年在考雅思 (IELTS) 的過程，口說跟寫作我想要達到 7 分。但是口說練了很多次、找了兩位口說老師練習還是找不到明顯進步的方法，也已經翻閱幾本網路上許多人推薦的書籍。每個人的答案似乎都沒有給我一個一致的方向。

    解決方法

    我讓自己有固定的運動習慣。當覺得有點不確定接下來怎麼走的時候，我會去打羽球或做其他運動。我很享受在羽球場上棋逢對手那樣鬥智鬥力的過程：可以流汗增進腦內啡也可以轉換注意力。加上球場上比的不只是技術、也是 mindset ：「只要不覺得自己會輸給對方、自己已經有一定的實力基礎一定可以打出一場精彩的球」。

    所以我當時抽離自己「焦急」的狀態，跟自己說：「先轉換一下心情放鬆一下、你可以的。」做一下其他事、或者打完球、讓情緒平穩後，重新看了IELTS口說評分、對照一些台灣大陸雅思口語名師的分享視頻，我知道自己在「發音」上跟練習的量要提高。相關話題的字詞（collocation）可以整理在一起，並且用自問自答的方式做延伸。這樣的方式同時訓練 part1, part2, part3 的口說。當知道該怎麼做、剩下的就是刻意大量的練習來累積，讓時間去展現成果。當有一個努力方向跟做法的時候、「不知從何施力的焦急」就平復了。

    同樣的在程式設計、 coding 也類似。這對我來說是新的學習領域、會有新的學習曲線。我剛接觸 git 的時候好像了解了，但對於 branch 切換等等， add、commit、push 等也不熟悉。但跟著 Discord 的討論社群、還有不同範例做了幾次，一點一點的就這樣比一個月前的我熟悉了。除了自己找答案跟重複的做之外、找到有專業有經驗的人點一下，也會加快自己的學習速度。

  • Q7 請用簡短兩三句話，描述對你而言，最重要的一項人際交往原則。

    • 共好

    意思是我隨著年紀，越重視常相處的人是不是都有隨時間在 update 自己，讓自己更好、也有形無形的鼓舞周圍的人。這些 update 可以是心態、能力技術、經驗等各方面。

    因為當一個人始終有目標而且正向地進步的時候，他/她是散發光芒的，彼此在這樣的交流下、雙方都會互相激勵各自的目標、同時從對方的思維學習，形成一個正循環。

    也類似「WeHelp」 這個 term ，因為每個人彼此努力、也彼此幫助，這個能量可以更擴散出去、會發揮更大的力量。
